06 / Safety + autonomy

Autonomous
without being
reckless.

Persistence is not permission. The loop can keep trying inside the product scope, but it does not silently expand authority, publish externally, spend without bounds, or invent missing product decisions.

SECRET BOUNDARY

Keys stay local

The key is read from a hidden prompt or environment variable, stored owner-only, referenced symbolically by Pi, and excluded from dashboard APIs and project artifacts.

PROJECT BOUNDARY

Worktrees isolate change

Each parallel worker gets a bounded task and separate checkout. The parent reviews evidence before integration.

QUALITY BOUNDARY

No self-certified shipping

Implementation and final evaluation are separated. User-facing work requires UI evidence, not merely passing unit tests.

STOP BOUNDARY

Real blockers are explicit

The run pauses only for missing authority, repeated external failure, impossible acceptance, or a decision that would materially change the product.
