02 / Architecture

A package,
not a pet repo.

The system has five deliberately small layers. There is no database, queue, SaaS dashboard, or custom agent runtime. Pi remains the runtime; Git remains the isolation mechanism; JSON and logs remain the observable truth.

LAYER A

CLI

product-loop supplies setup, init, start, status, validate, logs, dashboard, and doctor commands.

LAYER B

Pi skill

SKILL.md tells the parent agent how to clarify, simplify, delegate, review, integrate, and stop.

LAYER C

Pi subagents

Parent, workers, and blind reviewer use the models actually returned by your proxy. Workers run in isolated Git worktrees.

LAYER D

State protocol

.loop/run-state.json is the resumable machine state. Evidence files and logs explain why it changed.

LAYER E

Local dashboard

A zero-dependency Node server renders this manual and polls safe, read-only project endpoints every three seconds.

BOUNDARY

Your product repo

The package stays external. A project receives only PRD.md, local Pi settings, runtime state, and normal product code.

Why this is faster

Discovery once

The proxy registry is queried once during setup; model roles are selected automatically instead of hand-configuring every project.

Parallel attempts

Cheap capable workers explore and implement independent paths while expensive models focus on orchestration and judgment.

Failures become data

Every rejected phase produces evidence and feedback, so retries target the defect rather than restarting the entire project.
